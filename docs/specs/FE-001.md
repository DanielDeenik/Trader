## Problem

The existing `frontend/` is a thin React shell with 9 generic pages. It does not match the **fiscal.ai-inspired workflow UI** Dan described: a left-rail navigation organized by the 5-layer cognitive topology, a top bar surfacing live signal status, a main content area showing the workflow pipeline + ticker grid + per-ticker drill-down, and a right-rail HITL question queue where agents ask Dan to approve / defer / reject decisions.

We need wireframes (mid-fidelity, static HTML) that lock down information architecture and design tokens **before** any production React work begins. Implementing first and re-designing later is what produced the existing thin frontend.

## Goals

1. Define design tokens (colors, typography, spacing, radius, shadows) once and reuse across all wireframes.
2. Inventory the component primitives needed across the 5 views.
3. Produce 5 openable HTML wireframes that demonstrate the layout for each primary view.
4. Use realistic placeholder data (one ticker — AMD — fully populated per the 2026-04-21 daily brief shape).
5. Get Dan's per-view sign-off before any production React lands.

## Navigation model

```
[Top bar — global controls]
[Left rail]   [Main]                                      [Right rail]
 L0 Infra      <view content>                               HITL queue
 L1 Signals                                                  (collapsible)
 L2 Mosaics
 L3 Theses
 L4 Decisions
 L5 Portfolio
 ─────
 Settings
```

- Left rail: layer-organized nav. Each layer expands to show its sub-views.
- Top bar: persistent across all views.
- Main: swaps based on left-rail selection.
- Right rail: collapsible drawer (per [DLOG-9](https://www.notion.so/34e7772fa7ae81bb8c1bdb6c105cb3a5)). Shows the HITL queue regardless of which main view is active.

## The 5 primary views

### V1 — Shell (`v1-shell.html`)

Empty main content, but full chrome: top bar, left rail expanded, right rail expanded with 3 sample HITL cards. Demonstrates the layout grid + token usage.

### V2 — Workflow Pipeline (`v2-workflow-pipeline.html`)

Horizontal flow showing the 5-layer pipeline as connected stage cards. Each stage shows:
- Icon + layer name
- Live counter (e.g. "480 signals collected today")
- Status pill (`active` / `idle` / `error`)
- Sparkline of last 24h volume

Below the pipeline: a "today's activity" feed with timestamped events from each layer.

### V3 — Ticker Grid (`v3-ticker-grid.html`)

The fiscal.ai-style data-dense table view. Columns:

| Symbol | Divergence | Coherence | Lifecycle | Kelly % | 24h sparkline | Last action | Actions |
|---|---|---|---|---|---|---|---|
| AMD | 70.4 | 100 | Validating | 0.25 | ~~~ | build_thesis 16h ago | View · Promote · Defer |

Shows ~20 rows of mock tickers (AMD with the 2026-04-21 brief's actual numbers). Sortable, filterable. AMD row clickable (navigates to a per-ticker drill-down — out of scope for V3, just demonstrate the click affordance).

### V4 — HITL Queue (full-page) (`v4-hitl-queue.html`)

Same questions as the right-rail drawer but in a full-page list view with more context per card. Each card shows:
- Question header (e.g. "Approve build_thesis on AMD?")
- Context block: divergence, coherence, lifecycle, recent signals
- Provenance: which agent asked, on what data, when
- Action buttons: Approve · Defer · Reject
- Reason text input (required for Defer / Reject)

### V5 — Knowledge Graph (`v5-knowledge-graph.html`)

Cytoscape.js canvas centered on AMD as the focal node. Surrounding nodes:
- Connected tickers (NVDA, INTC) via "competitor" edges
- People (Lisa Su, Jensen Huang) via "leadership" edges
- Sectors (Semiconductors, AI Compute) via "belongs to" edges
- Recent signals (5–8) as derived nodes

Node styling by type, edge styling by relationship type. Click a node → side panel shows entity details + outbound relationships.

Layout: cose-bilkent (force-directed). Static positions on load — no live updates.
