## Problem

Social Arb ingests automated signals (yfinance, reddit, sec_edgar, …) but has no way to bring **user-curated unstructured sources** into the same substrate. Dan wants to drop a PDF analyst report, a competitor 8-K URL, or pasted text into a notebook and have those sources become first-class citizens — searchable, embeddable, and citable from cited-RAG output.

This ticket builds the **storage and parsing primitive** that all downstream notebook capability depends on (NB-002..006).

## Goals

1. A `Source` entity with stable IDs that any layer can reference.
2. Parsers for **PDF, DOCX, HTML/URL, plain text, pasted text** behind a single `ParserProtocol` interface.
3. A token-aware chunker that produces deterministic `SourceChunk` records.
4. An `EmbedderProtocol` with two implementations: `sentence-transformers` (local, free) and Voyage AI (finance-tuned).
5. A `SourceStore` API that supports: `save`, `get`, `delete`, `list`, `search_similar(query, k)`.
6. A migration that creates `sources` + `source_chunks` tables with a pgvector index for production; falls back to sqlite + Python cosine similarity for tests/dev.

### Key contracts

```python
class ParserProtocol(Protocol):
    def parse(self, blob: bytes | str, hint: ContentHint) -> ParsedSource: ...

class EmbedderProtocol(Protocol):
    dim: int
    def embed_batch(self, texts: list[str]) -> list[list[float]]: ...

class SourceStore(Protocol):
    def save(self, source: Source, chunks: list[ChunkWithVector]) -> str: ...
    def get(self, source_id: str) -> Source: ...
    def search_similar(self, query: str, k: int = 10, filter_source_ids: list[str] | None = None) -> list[SearchHit]: ...
```

## Acceptance criteria

- [ ] `from social_arb.sources import Source, SourceChunk` succeeds
- [ ] All 4 parsers (`text`, `pdf`, `docx`, `url`) round-trip a fixture in tests
- [ ] `Chunker` produces deterministic chunk IDs (same input ⇒ same IDs)
- [ ] `EmbedderProtocol` interface used by both sentence-transformers and Voyage impls
- [ ] `SqliteStore` (default in tests) stores Sources + chunks + vectors and retrieves top-k by cosine
- [ ] `PgvectorStore` connection string is recognized and SQL is valid (no live Postgres needed for the test suite — just a SQL-construction test)
- [ ] Roundtrip test: parse fixture → chunk → embed → save → search returns the same chunk in top-1
- [ ] No new dependency conflicts (`pip install ".[cloud]"` still works)
- [ ] Unit + integration tests added under `tests/sources/` and `tests/notebooks/`
